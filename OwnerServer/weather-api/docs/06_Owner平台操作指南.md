## 概述

本指南详细说明如何将 Weather API 部署到通用 API 服务平台，供 Developer 调用。

**前提条件**：
- Weather API 已在本地 8001 端口运行
- API 平台已在 8000 端口运行
- 您拥有 Owner 角色账号

---

## 第一步：登录 Owner Portal

### 1.1 访问控制台

1. 打开浏览器，访问：`http://localhost:3000`（前端地址）
2. 使用 Owner 账号登录
3. 确认右上角显示 **"仓库所有者"** 角色

### 1.2 进入仓库管理

1. 点击左侧菜单 **"仓库管理"**
2. 页面显示当前仓库数量：**我的仓库 0**
3. 点击右上角蓝色按钮 **"+ 创建仓库"**

---

## 第二步：创建 API 仓库

### 2.1 填写仓库基本信息

点击 **"+ 创建仓库"** 按钮后，会弹出创建仓库弹窗。在 **"基本信息"** Tab 中填写：

| 字段 | 填写内容 | 说明 |
|------|----------|------|
| **仓库标识** | `weather-api` | 英文，唯一标识 |
| **显示名称** | `天气 API` | 中文，显示给 Developer |
| **描述** | 提供实时天气、天气预报、空气质量查询服务 | 简短描述功能 |
| **仓库类型** | 自定义 | 选择仓库类型 |
| **协议类型** | HTTP | 选择协议类型 |
| **API端点地址** | `http://localhost:8001` | Weather API 服务地址 |

### 2.2 配置 API 端点

创建仓库后，点击 **"编辑"** 按钮，进入仓库配置页面：

1. 切换到 **"端点配置"** Tab
2. 点击 **"添加端点"** 按钮
3. 依次添加以下 4 个端点：

#### 端点配置列表

| 端点名称 | 方法 | 路径 | 描述 | RPM限制 |
|----------|------|------|------|--------|
| 实时天气 | GET | `/api/v1/weather/current` | 获取当前天气 | 1000 |
| 天气预报 | GET | `/api/v1/weather/forecast` | 获取天气预报 | 1000 |
| 空气质量 | GET | `/api/v1/weather/aqi` | 获取AQI数据 | 1000 |
| 天气预警 | GET | `/api/v1/weather/alerts` | 获取预警信息 | 500 |

**添加端点步骤**：
1. 点击 **"添加端点"** 按钮
2. 在弹窗中填写端点信息：
   - **HTTP方法**：选择 `GET`
   - **端点路径**：如 `/api/v1/weather/current`
   - **描述**：如 `获取当前天气`
   - **分类**：如 `weather`
   - **RPM限制**：可选，默认1000
3. 点击 **"保存"** 按钮

### 2.3 配置限流策略

在 **"限流配置"** Tab 中配置访问频率限制：

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| **每分钟请求数 (RPM)** | 1000 | 单个API Key每分钟最大请求数 |
| **每小时请求数 (RPH)** | 10000 | 单个API Key每小时最大请求数 |
| **每日请求数 (RPD)** | 100000 | 单个API Key每日最大请求数 |
| **突发限制** | 100 | 允许的突发请求数 |
| **并发限制** | 10 | 允许的最大并发请求数 |
| **请求超时 (秒)** | 30 | 请求超时时间 |

配置完成后，点击 **"保存限流配置"** 按钮。

### 2.4 保存仓库

1. 填写完基本信息后，点击 **"创建"** 按钮
2. 系统提示仓库创建成功
3. 仓库状态为 **"待审核"**

### 2.3 Weather API 接口参考

创建仓库后，可在仓库详情中添加以下接口：

| 接口名称 | 请求路径 | 方法 | 描述 |
|----------|----------|------|------|
| 实时天气 | `/weather/current` | GET | 获取当前天气 |
| 天气预报 | `/weather/forecast` | GET | 获取天气预报 |
| 空气质量 | `/weather/aqi` | GET | 获取AQI数据 |
| 天气预警 | `/weather/alerts` | GET | 获取预警信息 |

**通用请求参数**：

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| city | string | 是 | 城市名称 |

### 2.4 保存仓库

1. 点击 **"保存"** 或 **"创建"** 按钮
2. 系统提示仓库创建成功
3. 进入仓库详情页进行后续配置

## 第三步：编辑仓库配置端点

如果创建仓库时未配置端点，可以在仓库列表中点击 **"编辑"** 按钮进行配置：

1. 点击仓库列表中 `weather-api` 右侧的 **"编辑"** 按钮
2. 切换到 **"端点配置"** Tab
3. 点击 **"添加端点"** 按钮
4. 依次添加 4 个端点（参考 2.2 节的内容）
5. 完成后点击 **"保存配置"** 按钮

**编辑仓库功能说明**：

| Tab | 功能 | 说明 |
|-----|------|------|
| 基本信息 | 仓库基本配置 | 可修改显示名称、描述、API地址等 |
| 端点配置 | API端点管理 | 添加/编辑/删除 API 接口 |
| 限流配置 | 访问频率限制 | 设置 RPM/RPH/RPD 等限制 |

## 第四步：配置定价策略

> ⚠️ **注意**：定价配置功能正在开发中，目前可以使用默认的免费定价。如果界面中有定价配置选项，可以按需设置。

### 4.1 定价模式说明

平台支持以下计费模式：

| 计费模式 | 说明 | 适用场景 |
|----------|------|----------|
| **按次计费** | 每次 API 调用收费 | 简单 HTTP API |
| **按Token计费** | 根据 Token 数量收费 | AI/大模型 API |
| **按流量计费** | 根据流量收费 | 文件传输 API |
| **订阅计费** | 按月/年订阅 | SaaS 服务 |
| **免费** | 不收费 | 测试/公开 API |

### 4.2 建议的定价策略

对于 Weather API，建议使用 **免费** 模式或 **按次计费** 模式：

| 方案 | 定价 | 描述 |
|------|------|------|
| 方案一 | 免费 | 完全免费使用 |
| 方案二 | 按次计费 | ¥0.01/次 |

## 第五步：等待审核上线

### 5.1 审核状态说明

仓库创建后需要管理员审核，状态流转：

```
待审核 → 已审核 → 已上线
   ↓
已拒绝（需修改后重新提交）
```

### 5.2 查看审核进度

1. 在仓库列表中查看状态
2. 点击仓库行右侧的 **"详情"** 按钮查看审核意见
3. 如状态为 **"已拒绝"**，根据反馈修改后重新提交审核

---

## 第六步：上线运营

### 6.1 仓库上线

审核通过后：

1. 仓库状态变为 **"已上线"**
2. Developer 可以在 **"API 市场"** 中看到您的 API
3. Developer 可以订阅并调用您的 API

### 6.2 查看调用数据

1. 点击仓库列表右侧的 **"详情"** 按钮
2. 查看 **"API端点列表"** 和 **"限流配置"** 信息

### 6.3 查看收益结算

1. 点击左侧菜单 **"收益结算"**（如果可用）
2. 查看：
   - 本月收入
   - 累计收入
   - 结算记录
   - 提现申请

## 第七步：Developer 调用流程

### 7.1 Developer 订阅 API

Developer 的操作流程：

1. 登录 Developer Portal
2. 进入 **"API 市场"**
3. 找到 **"天气 API"**
4. 点击 **"订阅"**
5. 完成订阅后获得 **API Key**

### 7.2 Developer 调用示例

Developer 使用获得的 API Key 调用接口：

```bash
# 实时天气查询
curl -X GET "http://localhost:8000/api/v1/weather/current?city=北京" \
  -H "Authorization: Bearer {API_KEY}"

# 天气预报查询
curl -X GET "http://localhost:8000/api/v1/weather/forecast?city=北京&days=3" \
  -H "Authorization: Bearer {API_KEY}"
```

## 常见问题

### Q1: 为什么仓库显示"审核中"？

A: 新创建的仓库需要管理员审核，通常需要 1-2 个工作日。审核通过后自动上线。

### Q2: 如何修改已上线的仓库？

A: 
1. 点击仓库的 **"编辑"** 按钮
2. 修改配置后保存
3. 重新提交审核
4. 审核通过后更新生效

### Q3: Developer 调用报错 401 怎么办？

A: 
- 检查 API Key 是否正确
- 检查 API Key 是否过期
- 检查 Developer 是否已订阅该 API

### Q4: 如何查看具体的调用日志？

A:
1. 进入 **"数据分析"** → **"调用日志"**
2. 可以按时间、应用、接口筛选
3. 查看请求详情和响应数据

### Q5: 端点配置在哪里？

A:
1. 在仓库列表中点击仓库右侧的 **"编辑"** 按钮
2. 切换到 **"端点配置"** Tab
3. 可以添加、编辑、删除 API 端点

### Q6: 限流配置在哪里？

A:
1. 在仓库列表中点击仓库右侧的 **"编辑"** 按钮
2. 切换到 **"限流配置"** Tab
3. 可以设置 RPM/RPH/RPD/突发限制/并发限制等参数
4. 修改后点击 **"保存限流配置"** 按钮

### Q7: 收入如何结算？

A:
- 按月结算
- 每月 1 日生成上月账单
- 扣除平台服务费（10%）后转入您的账户
- 支持银行卡或支付宝提现

## 附录：配置文件参考

### Weather API 启动脚本

创建 `start_weather_api.bat`（Windows）：

```batch
@echo off
cd /d D:\Work_Area\AI\API-Agent\OwnerServer\weather-api

:: 激活虚拟环境（如果使用）
:: call venv\Scripts\activate

:: 启动服务
echo 正在启动 Weather API...
echo 访问地址: http://localhost:8001
echo API 文档: http://localhost:8001/docs
uvicorn src.main:app --host 0.0.0.0 --port 8001 --reload

pause
```

### API 平台启动脚本

创建 `start_api_platform.bat`（Windows）：

```batch
@echo off
cd /d D:\Work_Area\AI\API-Agent\api-platform

:: 启动后端
echo 正在启动 API 平台后端...
start uvicorn src.main:app --reload --host 0.0.0.0 --port 8000

:: 启动前端
cd web
echo 正在启动 API 平台前端...
start npm run dev

echo.
echo API 平台已启动:
echo - 前端: http://localhost:3000
echo - 后端: http://localhost:8000
echo.
pause
```
