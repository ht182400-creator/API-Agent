## Owner 角色说明

### Owner 能做什么？

| 功能 | 描述 |
|------|------|
| 创建 API 产品 | 在平台上创建和管理自己的 API 产品 |
| 开发 API | 使用平台提供的框架开发 API |
| 部署上线 | 将 API 部署到平台并发布 |
| 定价收费 | 设置 API 定价策略并收取费用 |
| 数据分析 | 查看 API 调用统计和收入报表 |
| 版本管理 | 管理 API 版本，支持回滚 |

### Owner 不能做什么？

- 不能访问 Developer 的数据
- 不能修改平台核心配置
- 不能访问其他 Owner 的 API

## Weather API 示例

Weather API 是一个完整的示例项目，展示了如何开发一个天气查询 API 产品。

### 快速开始

```bash
cd weather-api

# 安装依赖
pip install -r requirements.txt

# 如果启动时报 ModuleNotFoundError，手动安装缺失的依赖
pip install loguru pydantic-settings

# 运行测试
pytest tests/ -v

# 启动服务（使用 8001 端口，避免与 API 平台冲突）
uvicorn src.main:app --reload --port 8001

# 访问 API
curl http://localhost:8001/api/v1/weather/current?city=北京
```

### API 接口

| 接口 | 方法 | 描述 |
|------|------|------|
| `/api/v1/weather/current` | GET | 实时天气查询 |
| `/api/v1/weather/forecast` | GET | 天气预报查询 |
| `/api/v1/weather/aqi` | GET | 空气质量查询 |
| `/api/v1/weather/alerts` | GET | 天气预警查询 |

## 开发规范

### 代码规范

- 遵循 PEP 8（Python）
- 使用 TypeScript 类型定义（前端）
- 添加 JSDoc 注释
- 行内注释使用中文

### API 设计规范

- RESTful 风格
- 版本控制：`/api/v1/`
- 统一响应格式
- 完善的错误处理

### 安全规范

- API Key 认证
- 参数校验
- 限流保护
- 日志审计

## 常见问题

### Q: 如何成为 Owner？

A: 在平台上注册账号后，申请成为 API Provider，经审核后获得 Owner 权限。

### Q: 可以开发哪些类型的 API？

A: 可以开发任何类型的 API，只要符合平台规范和法律要求。常见类型包括：
- 数据查询类（天气、地图、金融数据）
- AI 服务类（图像识别、自然语言处理）
- 工具类（短信、邮件、推送）
- 内容类（新闻、图片、视频）

### Q: 如何定价 API？

A: 平台支持多种定价模式：
- 免费版：限制调用次数
- 按量付费：按调用次数计费
- 订阅制：按月/年收费
- 定制套餐：根据用户需求定制

### Q: 收入如何结算？

A: 平台按月结算收入，扣除平台服务费后转入您的账户。具体费率请参考平台定价政策。
