## 1. 概述

### 1.1 项目背景

随着 API 平台业务的快速发展，平台上的 API 仓库数量持续增长。当前的 Developer 测试工具（`CallWeatherTest Tool`）仅支持天气 API，无法满足日益增长的需求。

本项目旨在开发一个**可扩展的统一 API 测试工具**，支持：
- 多仓库切换
- 动态表单生成
- 分类展示（自研/第三方）
- 请求历史记录

### 1.2 项目目标

| 目标 | 描述 | 成功指标 |
|------|------|----------|
| **可扩展性** | 新增仓库仅需添加配置文件 | 添加新仓库 < 30分钟 |
| **易用性** | 统一的测试界面 | 学习成本 < 5分钟 |
| **完整性** | 支持主流 API 类型 | 覆盖 95% 常用场景 |
| **可维护性** | 清晰的代码结构 | 新人可独立维护 |

### 1.3 术语表

| 术语 | 定义 |
|------|------|
| **仓库 (Repository)** | API 服务的集合，如 Weather API、Payment API |
| **端点 (Endpoint)** | API 仓库中的具体接口，如 /weather/current |
| **自研仓库** | 平台自主研发的 API 服务 |
| **第三方仓库** | 外部接入的 API 服务 |
| **动态表单** | 根据配置自动生成的表单，无需硬编码 |

## 3. 功能需求

### 3.1 功能列表

### 3.2 功能详细说明

#### F-001 仓库选择

**描述**：用户可以从仓库列表中选择要测试的 API 仓库

**交互流程**：
1. 页面加载时展示所有已配置的仓库
2. 用户点击仓库卡片进行选择
3. 选中仓库后高亮显示
4. 右侧端点列表自动更新

**数据来源**：`repositories.config.ts`

**状态**：
- 未选中：无高亮
- 已选中：蓝色边框 + 背景色 + 勾选图标

#### F-002 分类筛选

**描述**：用户可以按分类筛选显示的仓库

**分类定义**：
| 分类 | 标识 | 说明 |
|------|------|------|
| 全部 | all | 显示所有仓库 |
| 自研 | self | 显示平台自研仓库 |
| 第三方 | third_party | 显示第三方接入的仓库 |

**交互**：
- 点击 Tab 切换分类
- 当前选中 Tab 高亮显示
- 实时更新仓库列表

#### F-004 动态表单

**描述**：根据端点的参数配置自动生成表单

**支持的参数类型**：
| 类型 | 渲染控件 | 说明 |
|------|----------|------|
| string | 文本输入框 | 普通字符串输入 |
| number | 数字输入框 | 数值输入，支持 min/max |
| boolean | 复选框 | true/false 开关 |
| select | 下拉选择 | 预设选项列表 |

**参数位置**：
| 位置 | 说明 |
|------|------|
| query | URL 查询参数 |
| path | URL 路径参数 |
| header | 请求头参数 |
| body | 请求体参数 |

#### F-007 响应展示

**描述**：展示 API 响应的详细信息

**展示内容**：
- HTTP 状态码和状态文本
- 响应耗时（毫秒）
- 响应数据（JSON 格式化）
- 错误信息（如有）

**状态码颜色**：
| 状态码范围 | 颜色 | 含义 |
|------------|------|------|
| 200-299 | 绿色 | 成功 |
| 300-399 | 蓝色 | 重定向 |
| 400-499 | 黄色 | 客户端错误 |
| 500+ | 红色 | 服务器错误 |
