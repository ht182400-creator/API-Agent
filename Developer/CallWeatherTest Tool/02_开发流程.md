# Weather API 调用测试工具 - 开发流程

## 1. 项目概述

### 1.1 项目目标

快速构建一个轻量级的 Web 测试工具，用于调用 API 平台的天气服务接口。

### 1.2 技术选型

| 技术 | 选择 | 理由 |
|------|------|------|
| 前端框架 | 原生 HTML/CSS/JS | 无需构建工具，快速部署 |
| HTTP 客户端 | Fetch API | 浏览器原生支持 |
| 样式框架 | Tailwind CSS (CDN) | 现代化 UI，快速上手 |
| 图标 | Heroicons (CDN) | 简洁美观 |
| 数据存储 | LocalStorage | 持久化 API Key |

### 1.3 项目结构

```

├── css/
```

## 4. 接口对接

### 4.1 API 基础信息

| 配置项 | 值 |
|--------|-----|
| Base URL | `http://localhost:8000/api/v1` |
| 认证方式 | Bearer Token |
| Content-Type | `application/json` |

### 4.2 接口列表

| 接口 | 方法 | 路径 | 描述 |
|------|------|------|------|
| 实时天气 | GET | `/weather/current` | 获取当前天气 |
| 天气预报 | GET | `/weather/forecast` | 获取天气预报 |
| 空气质量 | GET | `/weather/aqi` | 获取 AQI 数据 |
| 天气预警 | GET | `/weather/alerts` | 获取预警信息 |

### 4.3 调用示例

```javascript
// 实时天气查询
async function getCurrentWeather(city) {
  const response = await fetch(
    `${API_BASE}/weather/current?city=${encodeURIComponent(city)}`,
    {
      headers: {
        'Authorization': `Bearer ${API_KEY}`,
        'Content-Type': 'application/json'
      }
    }
  );
  return await response.json();
}
```

## 6. 错误处理

### 6.1 错误类型

| 错误码 | 说明 | 用户提示 |
|--------|------|----------|
| 401 | 未授权 / API Key 无效 | 请检查 API Key 是否正确 |
| 403 | 权限不足 | 当前 Key 无权访问此接口 |
| 404 | 接口不存在 | 接口地址有误，请联系管理员 |
| 429 | 请求过于频繁 | 请稍后再试 |
| 500 | 服务器内部错误 | 服务端异常，请稍后重试 |
| network | 网络错误 | 网络连接失败，请检查网络 |

### 6.2 错误展示

- 红色提示框显示错误信息
- 详细错误码和描述
- 提供重试按钮
