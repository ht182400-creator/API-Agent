# Weather API 调用测试工具

## 产品概述

Weather API 调用测试工具是一款面向开发者的 Web 应用，用于测试和验证 API 平台天气服务的可用性和响应格式。

## 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| API Key 配置 | 设置和管理 API 密钥 | P0 |
| 实时天气查询 | 测试 `/api/v1/weather/current` 接口 | P0 |
| 天气预报查询 | 测试 `/api/v1/weather/forecast` 接口 | P0 |
| 空气质量查询 | 测试 `/api/v1/weather/aqi` 接口 | P1 |
| 天气预警查询 | 测试 `/api/v1/weather/alerts` 接口 | P1 |
| 响应格式化 | 美化 JSON 响应展示 | P1 |
| 调用历史记录 | 记录最近 10 次 API 调用 | P2 |

## 用户需求

### 目标用户

- **初级开发者**：首次使用 API 平台，希望快速验证 API 可用性
- **测试工程师**：需要批量测试天气 API 的各种场景
- **集成开发者**：在正式集成前验证接口参数和响应格式

### 用户场景

1. **场景一**：开发者刚获得 API Key，需要验证 Key 是否有效
2. **场景二**：测试不同城市参数的返回结果
3. **场景三**：查看 API 响应格式，便于编写解析代码
4. **场景四**：对比不同仓库/版本的 API 响应差异

## 功能需求详情

### F1: API Key 配置

- 输入框用于填写 API Key
- 本地存储 API Key，避免重复输入
- 支持切换不同环境的 API 地址
- 显示 Key 的基本信息（所属仓库、权限范围）

### F2: 实时天气查询

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| city | string | 是 | 城市名称（中文或拼音） |

**响应字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| temperature | number | 温度（摄氏度） |
| humidity | number | 湿度（%） |
| weather | string | 天气状况 |
| wind_speed | number | 风速（m/s） |
| update_time | string | 更新时间 |

### F3: 天气预报查询

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| city | string | 是 | 城市名称 |
| days | number | 否 | 预报天数（默认7天） |

### F4: 空气质量查询

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| city | string | 是 | 城市名称 |

### F5: 天气预警查询

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| city | string | 是 | 城市名称 |

## 非功能需求

### 性能需求

- 页面首次加载时间 < 2秒
- API 请求超时时间 30 秒
- 本地存储容量 5MB

### 兼容性需求

- 支持 Chrome、Firefox、Safari、Edge 最新两个版本
- 响应式设计，适配 1024px 以上屏幕

### 安全需求

- API Key 仅存储在浏览器本地
- 不向第三方服务器发送用户数据
- 支持 HTTPS 传输

## 验收标准

| ID | 验收条件 | 测试方法 |
|----|----------|----------|
| AC1 | 输入有效 API Key 后可正常调用天气接口 | 输入 Key，调用接口，验证返回数据 |
| AC2 | 输入无效 API Key 时显示友好错误提示 | 输入错误 Key，验证错误信息 |
| AC3 | 所有接口响应以格式化 JSON 展示 | 查看响应区域，验证格式 |
| AC4 | 调用历史自动记录并可清空 | 调用接口后检查历史列表 |
| AC5 | API Key 刷新页面后保持 | 刷新页面后检查 Key 是否保留 |
