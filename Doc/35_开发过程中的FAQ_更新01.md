### 2.2 问题二：集成测试工具调用路径错误

#### 问题描述
集成测试工具调用API后，响应返回200但内容为空，显示`{"detail": "Not Found"}`

#### 问题原因
1. **前端URL构建错误**：前端构建的URL是`/weather-api/current`，但后端proxy接口完整路径是`/api/v1/repositories/{repo_slug}/{path}`
2. **路由匹配问题**：`/{repo_slug}`路由在`/{repo_slug}/{path:path}`之前定义，导致路径被错误匹配
3. **仓库endpoint_url未配置**：数据库中Weather仓库的`endpoint_url`为NULL

#### 解决方案

**步骤1：修复前端URL构建逻辑**

修改 `ApiTester.tsx` 中的 `buildUrl` 函数：

```typescript
// 修改前 - 错误的URL格式
const buildUrl = useCallback((repo: Repository, endpoint: Endpoint): string => {
    // ...
    // 通过 API Platform 后端 proxy 接口调用：/{repo_slug}/{path}
    return `/${repo.slug}${path}${queryString ? '?' + queryString : ''}`;
}, [paramValues]);

// 修改后 - 正确的URL格式
const buildUrl = useCallback((repo: Repository, endpoint: Endpoint): string => {
    // ...
    // 通过 API Platform 后端 proxy 接口调用：/api/v1/repositories/{repo_slug}/{path}
    return `/api/v1/repositories/${repo.slug}${path}${queryString ? '?' + queryString : ''}`;
}, [paramValues]);
```

**步骤2：修复仓库配置**

修改 `weather.config.ts` 中的 `baseUrl` 和 `apiUrl`：

```typescript
export const weatherRepository: Repository = {
    id: 'weather-api',
    slug: 'weather-api',  // 与数据库一致
    // ...
    baseUrl: '/api/v1/repositories/weather-api',
    apiUrl: 'http://localhost:8000',  // 通过API Platform后端调用
};
```

**步骤3：修复后端路由匹配**

修改 `repositories.py` 中的 `get_repository` 函数，添加路径检查防止错误匹配：

```python
@router.get("/{repo_slug}", response_model=BaseResponse[RepositoryResponse])
async def get_repository(
    repo_slug: str,
    request: Request,
    db: AsyncSession = Depends(get_db),
):
    # 防止 /{repo_slug} 路由错误匹配带有额外路径的请求
    # 如 /repositories/weather-api/current 应该匹配 proxy 路由，而不是这个路由
    if "/" in repo_slug:
        raise HTTPException(status_code=404, detail="Repository not found")
    
    # 继续原有逻辑...
```

**步骤4：更新数据库仓库配置**

```sql
UPDATE repositories 
SET endpoint_url = 'http://localhost:8001/api/v1/weather'
WHERE slug = 'weather-api';
```

## 遇到的问题与解决

### 5.1 问题汇总表

| 序号 | 问题描述 | 根本原因 | 解决方案 | 状态 |
|------|----------|----------|----------|------|
| 1 | 消费明细显示空数据 | 数据库缺少字段 | 执行迁移脚本添加字段 | ✅ 已解决 |
| 2 | API调用返回404 | URL路径不正确 | 修改前端URL构建逻辑 | ✅ 已解决 |
| 3 | 路由匹配错误 | 路由顺序问题 | 添加路径检查防止错误匹配 | ✅ 已解决 |
| 4 | 后端无法转发请求 | endpoint_url为NULL | 更新数据库配置 | ✅ 已解决 |
| 5 | 列标题不通用 | 标题过于特定 | 改为"调用者" | ✅ 已解决 |

### 5.2 经验总结

1. **URL路径规范**：前端调用后端proxy接口必须使用完整路径 `/api/v1/repositories/{slug}/{path}`
2. **路由定义顺序**：FastAPI按定义顺序匹配路由，带路径参数的路由应放在具体路由之后
3. **数据库配置同步**：前端仓库配置应与数据库保持一致，特别是`slug`字段
4. **迁移脚本规范**：修改数据库结构前应创建迁移脚本，便于环境同步

## Q58: 如何添加仓库到集成测试页面？

**问题描述**：想在API测试工具中添加新的仓库选项，应该如何操作？

**解决方案**：

### 步骤1：创建仓库配置文件

在 `web/src/config/repos/` 目录下创建新的配置文件，如 `example.config.ts`：

```typescript
import { Repository } from '../../../types/api-tester';

export const exampleRepository: Repository = {
  id: 'example-api',
  slug: 'example-api',  // 必须与数据库 repositories 表的 slug 一致
  name: '示例API',
  description: 'API描述',
  icon: 'api',
  category: 'self',  // self=自研, third=第三方
  baseUrl: '/api/v1/repositories/example-api',
  apiUrl: 'http://localhost:8000',  // API Platform 后端地址
  authType: 'api_key',
  authHeader: 'X-Access-Key',
  enabled: true,
  version: '1.0.0',
  endpoints: [
    {
      id: 'endpoint1',
      name: '示例接口',
      description: '接口描述',
      path: '/example',
      method: 'GET',
      params: [
        {
          name: 'param1',
          type: 'string',
          required: true,
          description: '参数描述',
          placeholder: '请输入',
          in: 'query'
        }
      ],
      tags: ['示例']
    }
  ]
};
```

### 步骤2：在仓库索引中注册

编辑 `web/src/config/repositories.config.ts`：

```typescript
import { exampleRepository } from './repos/example.config';

export const repositories = [
  exampleRepository,
  // ... 其他仓库
];
```

### 步骤3：确保数据库有对应记录

```sql
-- 在 repositories 表中添加记录
INSERT INTO repositories (id, name, slug, endpoint_url, status)
VALUES ('example-api', '示例API', 'example-api', 'http://目标服务地址', 'online');
```

### 步骤4：重启前端服务

刷新页面后，新仓库会出现在测试工具的选择框中。

**注意事项**：
1. `slug` 必须与数据库中 `repositories` 表的 `slug` 字段一致
2. `endpoint_url` 必须指向实际的后端服务地址
3. 如果是第三方API，`category` 应设置为 `third`

**相关文件**：
- 仓库配置示例：`api-platform/web/src/config/repos/weather.config.ts`
- 仓库索引文件：`api-platform/web/src/config/repositories.config.ts`
