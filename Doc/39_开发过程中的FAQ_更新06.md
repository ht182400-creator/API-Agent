## 问题150.27：开发者仓库预览和管理权限区分

**更新时间**：2026-04-24

### 150.27.1 需求描述

**用户反馈**：
1. **开发者应该可以预览所有仓库**（包括自建仓库和平台仓库）
2. **自建的仓库有管理权**（可以编辑、删除等）
3. **其他仓库只能预览详情**，知道如何调用这个仓库的API

### 150.27.2 问题分析

**当前问题**：
- `developerWithReposMenu` 只有"仓库管理"入口，没有"仓库市场"入口
- 开发者无法预览其他人的仓库
- 无法了解如何调用其他仓库的API

**需求目标**：
- 开发者可以预览所有仓库（仓库市场）
- 开发者可以管理自己的仓库（仓库管理）
- 开发者可以了解如何调用任意仓库的API

### 150.27.3 解决方案

#### 方案1：前端菜单调整

**修改文件**：`web/src/components/Layout.tsx`

**修改内容**：在有仓库的开发者菜单中同时显示"仓库市场"和"仓库管理"两个入口

```typescript
// 【V5.0更新】开发者菜单（有仓库版本）
// 【V5.0更新】同时显示仓库市场和仓库管理入口
const developerWithReposMenu: MenuProps['items'] = [
```

#### 方案2：路由权限配置

**修改文件**：`web/src/config/permissions.ts`

```typescript

```

#### 方案3：API调用指南

**修改文件**：`web/src/pages/developer/RepoDetail.tsx`

**修改内容**：新增 `ApiCallGuide` 组件，在仓库详情页面底部显示 API 调用指南

```tsx
// 【V5.0新增】API调用指南组件
const ApiCallGuide = ({ repo }: { repo: Repository }) => {
  const apiBaseUrl = '/api/v1'
  const repoSlug = repo.slug

  // 生成curl示例
  const generateCurlExample = (endpoint: RepositoryEndpoint) => {
    const url = `${apiBaseUrl}/repositories/${repoSlug}${endpoint.path}`
    return `curl -X ${endpoint.method} '${url}' \\
  -H 'Content-Type: application/json' \\
  -H 'X-Access-Key: YOUR_API_KEY' \\
  -H 'X-Signature: YOUR_SIGNATURE' \\
  -H 'X-Timestamp: TIMESTAMP' \\
  -H 'X-Nonce: NONCE'`
  }

  // 生成请求代码示例
  const generateCodeExample = (endpoint: RepositoryEndpoint, method: string) => {
    const url = `${apiBaseUrl}/repositories/${repoSlug}${endpoint.path}`
    if (method === 'GET') {
      return `fetch('${url}', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json',
    'X-Access-Key': 'YOUR_API_KEY',
    'X-Signature': 'YOUR_SIGNATURE',
    'X-Timestamp': 'TIMESTAMP',
    'X-Nonce': 'NONCE',
  }
})
  .then(res => res.json())
  .then(data => console.log(data))`
    } else {
      return `fetch('${url}', {
  method: '${method}',
  headers: {
    'Content-Type': 'application/json',
    'X-Access-Key': 'YOUR_API_KEY',
    'X-Signature': 'YOUR_SIGNATURE',
    'X-Timestamp': 'TIMESTAMP',
    'X-Nonce': 'NONCE',
  },
  body: JSON.stringify({
    // 您的请求参数
  })
})
  .then(res => res.json())
  .then(data => console.log(data))`
    }
  }

  return (
    <Card title="API调用指南">
      <Alert
        type="info"
        showIcon
        icon={<SafetyOutlined />}
        message="调用此API需要先获取API Key"
        description={
          <span>
            在 <Link to="/developer/keys">API Keys</Link> 页面创建和管理您的API Key。
          </span>
        }
      />
      <Collapse items={/* 每个端点的调用示例 */} />
    </Card>
  )
}
```

### 150.27.4 权限架构总结

```
┌───────────────────────────────────────────────────────────────────────┐
│                        仓库访问权限矩阵                                │
├──────────────┬─────────────┬─────────────┬─────────────┬─────────────┤
│    功能       │   管理员     │   仓库所有者   │    开发者    │   普通用户   │
├──────────────┼─────────────┼─────────────┼─────────────┼─────────────┤
│ 仓库市场      │     ✅       │     ✅       │     ✅       │     ✅       │
│ 仓库详情预览   │     ✅       │     ✅       │     ✅       │     ✅       │
│ 仓库管理      │     ✅       │     ✅       │     ✅       │     ❌       │
│  编辑仓库     │     ✅       │     ✅       │     ✅       │     ❌       │
│  删除仓库     │     ✅       │     ✅       │     ✅       │     ❌       │
│  端点配置     │     ✅       │     ✅       │     ✅       │     ❌       │
│  API调用     │     ✅       │     ✅       │     ✅       │     ✅       │
└──────────────┴─────────────┴─────────────┴─────────────┴─────────────┘
```

**说明**：
- **仓库市场**：所有用户都可以浏览所有在线仓库列表
- **仓库详情预览**：所有用户都可以查看仓库详细信息，了解API结构
- **仓库管理**：仅仓库创建者和管理员可以进行管理操作（编辑、删除、配置端点）
- **API调用**：所有有API Key的用户都可以调用已上线的API

### 150.27.6 验证方法

#### 仓库市场验证

1. 登录开发者账号（已有仓库的开发者）
2. 左侧菜单应该同时看到"仓库市场"和"仓库管理"两个入口
3. 点击"仓库市场" - 应该看到所有在线仓库列表（自建+平台）

#### 仓库管理验证

1. 点击"仓库管理" - 应该只看到自己创建的仓库
2. 可以编辑、删除自己的仓库
3. 无法编辑、删除其他人的仓库

#### API调用指南验证

1. 进入任意仓库详情页面（自建或平台仓库均可）
2. 页面底部应该显示"API调用指南"卡片
3. 可以看到每个端点的：
   - 调用地址
   - 请求头说明
   - Curl 示例代码
   - JavaScript 示例代码
