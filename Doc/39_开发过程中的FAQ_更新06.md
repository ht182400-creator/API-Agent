## 问题150.27：开发者仓库预览和管理权限区分

**更新时间**：2026-04-24

### 150.27.1 需求描述

**用户反馈**：
1. **开发者应该可以预览所有仓库**（包括自建仓库和平台仓库）
2. **自建的仓库有管理权**（可以编辑、删除等）
3. **其他仓库只能预览详情**，知道如何调用这个仓库的API

### 150.27.2 问题分析

**当前问题**：
- `developerWithReposMenu` 只有"仓库管理"入口，没有"仓库市场"入口
- 开发者无法预览其他人的仓库
- 无法了解如何调用其他仓库的API

**需求目标**：
- 开发者可以预览所有仓库（仓库市场）
- 开发者可以管理自己的仓库（仓库管理）
- 开发者可以了解如何调用任意仓库的API

### 150.27.3 解决方案

#### 方案1：前端菜单调整

**修改文件**：`web/src/components/Layout.tsx`

**修改内容**：在有仓库的开发者菜单中同时显示"仓库市场"和"仓库管理"两个入口


## 问题150.28：仓库详情页面白屏

**问题现象**：点击仓库详情页面时，页面显示白屏，控制台报错：

**修改内容**：将组件中使用的 `SafetyOutlined` 改为 `SafetyCertificateOutlined`：

```typescript
// 修改前（第 199 行）
icon={<SafetyOutlined />}

// 修改后

```

## 问题150.29：开发者仓库管理权限缺失

**更新时间**：2026-04-24

### 150.29.1 问题描述

**问题现象**：开发者点击仓库管理页面时，无法创建、编辑或删除仓库。

**问题原因**：`developer` 角色的默认权限中没有 `OWNER_REPO` 权限，而 `/owner/repos` 路由需要此权限。

### 150.29.2 问题分析

**权限配置检查**：
- `/owner/repos` 路由需要 `OWNER_REPO` 权限
- `developer` 角色的默认权限列表中不包含此权限
- 导致开发者无法访问仓库管理功能

### 150.29.3 解决方案

**修改文件**：`web/src/config/permissions.ts`

**修改内容**：为 `developer` 角色添加 `OWNER_REPO` 权限：

```typescript
// 修改前
developer: [
  Permission.DEV_API_KEYS,
  Permission.DEV_QUOTA,
  Permission.DEV_BILLING,
  Permission.LOG_READ,
  Permission.BILLING_READ,
  Permission.REPO_READ,
],

// 修改后
developer: [
  Permission.DEV_API_KEYS,
  Permission.DEV_QUOTA,
  Permission.DEV_BILLING,
  Permission.LOG_READ,
  Permission.BILLING_READ,
  Permission.REPO_READ,
  Permission.OWNER_REPO,  // 【V5.0新增】管理自己的仓库
],
```

### 150.29.4 相关文件修改清单

| 文件路径 | 修改内容 | 说明 |
|----------|----------|------|
| `web/src/config/permissions.ts` | 修改 | 为 developer 角色添加 OWNER_REPO 权限 |

### 150.29.5 验证方法

1. 登录开发者账号
2. 点击仓库管理页面
3. 可以正常创建、编辑、删除自己的仓库

## 问题150.30：开发者菜单缺少仓库管理入口

**更新时间**：2026-04-24

### 150.30.1 问题描述

**问题现象**：开发者的左侧菜单中没有"仓库管理"入口，只有"仓库市场"入口。

**问题原因**：
1. `getMenuItems` 函数根据 `userHasRepos` 状态选择菜单
2. 无仓库的开发者使用 `developerWithoutReposMenu`，没有仓库管理入口
3. `/user/has-repos` API 可能返回错误数据，导致 `hasRepos` 为 false

### 150.30.2 问题分析

**菜单配置检查**：
- `developerWithReposMenu`：有仓库管理入口 ✅
- `developerWithoutReposMenu`：没有仓库管理入口 ❌

**原代码逻辑**：
```typescript
case 'developer':
  if (userHasRepos) {
    return developerWithReposMenu  // 有仓库管理入口
  } else {
    return developerWithoutReposMenu  // 没有仓库管理入口
  }
```

### 150.30.3 解决方案

**修改文件**：`web/src/components/Layout.tsx`

**修改内容**：
1. 更新 `developerWithoutReposMenu`，添加仓库管理入口
2. 简化 `getMenuItems` 函数，统一使用相同菜单

```typescript
// 修改后的 developerWithoutReposMenu
// 【V5.0更新】默认显示仓库管理入口，API 会正确返回空列表
const developerWithoutReposMenu: MenuProps['items'] = [
  { key: '/', icon: <DashboardOutlined />, label: '工作台' },
  { key: '/developer/keys', icon: <KeyOutlined />, label: 'API Keys' },
  { key: '/developer/repos', icon: <ShopOutlined />, label: '仓库市场' },
  // 【V5.0更新】仓库管理入口 - 默认显示，让开发者能访问
  { key: '/owner/repos', icon: <FolderOutlined />, label: '仓库管理' },
  { key: '/developer/quota', icon: <PieChartOutlined />, label: '配额使用' },
  { key: '/developer/logs', icon: <FileTextOutlined />, label: '调用日志' },
  { key: '/developer/billing', icon: <WalletOutlined />, label: '账单中心' },
  { key: '/developer/recharge', icon: <DollarOutlined />, label: '充值中心' },
  { type: 'divider', label: '仓库所有者功能' },
  { key: '/owner/analytics', icon: <BarChartOutlined />, label: '数据分析' },
  { key: '/owner/settlement', icon: <DollarOutlined />, label: '收益结算' },
]

// 修改后的 getMenuItems
// 【V5.0更新】统一开发者菜单，所有开发者都能看到仓库管理入口
const getMenuItems = (userType: string, userHasRepos: boolean = false): MenuProps['items'] => {
  switch (userType) {
    case 'super_admin':
      return superAdminMenu
    case 'admin':
      return adminMenu
    case 'owner':
      return ownerMenu
    case 'developer':
      // 【V5.0更新】统一使用相同菜单，仓库管理入口默认显示
      return developerWithReposMenu
    case 'user':
    default:
      return userMenu
  }
}
```

### 150.30.6 功能说明

**仓库管理页面功能**：
- **创建仓库**：点击右上角"创建仓库"按钮
- **编辑仓库**：点击仓库行的"编辑"按钮
- **删除仓库**：点击仓库行的"删除"按钮（需确认）
- **查看详情**：点击仓库行的"详情"按钮
- **端点配置**：在编辑弹窗的"端点配置" Tab 中管理 API 端点
- **限流配置**：在编辑弹窗的"限流配置" Tab 中设置访问限制

**仓库创建后的流程**：
1. 创建仓库 → 状态为"待审核"
2. 管理员审核通过 → 状态为"已审核（待上线）"
3. 管理员上线仓库 → 状态为"已上线"
4. 上线后可在仓库市场中被其他用户发现和调用
