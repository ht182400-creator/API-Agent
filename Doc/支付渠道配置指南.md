# 支付渠道配置指南

## 概述

本文档说明如何为通用API服务平台配置真实的支付宝、微信支付等支付渠道。

---

## 支付模式说明

系统支持两种支付模式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **模拟模式** | 使用模拟支付页面，不产生真实交易 | 开发测试 |
| **真实模式** | 集成第三方支付SDK，产生真实交易 | 正式生产环境 |

---

## 支付配置 API

### 1. 获取支付配置状态

**接口**：`GET /api/v1/admin/payment-config/status`

**说明**：快速检查各支付渠道的启用状态和配置状态

**响应示例**：
```json
{
  "code": 0,
  "data": {
    "mock_mode": true,
    "channels": [
      {
        "channel": "alipay",
        "channel_name": "支付宝",
        "enabled": false,
        "sandbox": true,
        "configured": false,
        "status": "not_configured"
      },
      {
        "channel": "wechat",
        "channel_name": "微信支付",
        "enabled": false,
        "sandbox": true,
        "configured": false,
        "status": "not_configured"
      },
      {
        "channel": "bankcard",
        "channel_name": "银行卡",
        "enabled": false,
        "sandbox": false,
        "configured": false,
        "status": "not_configured"
      }
    ]
  }
}
```

### 2. 获取支付配置详情

**接口**：`GET /api/v1/admin/payment-config/detail`

**说明**：获取完整的支付配置（密钥信息脱敏）

### 3. 更新支付配置

**接口**：`PUT /api/v1/admin/payment-config/update`

**说明**：更新支付渠道配置，支持部分更新

**请求示例**：
```json
{
  "mock_mode": false,
  "alipay": {
    "enabled": true,
    "sandbox": false,
    "app_id": "2021001234567890",
    "private_key": "MIIEvQIBADANB...",
    "alipay_public_key": "MIIBIjANB...",
    "notify_url": "https://yourdomain.com/api/v1/payments/alipay/callback",
    "return_url": "https://yourdomain.com/recharge"
  }
}
```

### 4. 测试配置连通性

**接口**：
- `POST /api/v1/admin/payment-config/test/alipay`
- `POST /api/v1/admin/payment-config/test/wechat`

## 支付宝配置指南

### 前置条件

1. 注册蚂蚁金服开放平台账号：https://open.alipay.com/
2. 创建应用并获取 AppID
3. 配置 RSA2 密钥对

### 配置步骤

#### 1. 获取 AppID

登录蚂蚁金服开放平台 → 开发设置 → 获取 AppID

#### 2. 生成 RSA2 密钥

**工具推荐**：支付宝密钥生成工具

**命令生成（Linux/Mac）**：
```bash
# 生成私钥
openssl genrsa -out app_private_key.pem 2048

# 生成公钥
openssl rsa -in app_private_key.pem -pubout -out app_public_key.pem

# 转换为 PKCS8 格式（支付宝要求）
openssl pkcs8 -topk8 -inform PEM -outform PEM -in app_private_key.pem -out app_private_key_pkcs8.pem -nocrypt
```

#### 3. 提交公钥

在蚂蚁金服开放平台 → 应用详情 → 开发设置 → 接口加签方式 → 配置 RSA2(SHA256) 公钥

#### 4. 获取支付宝公钥

配置公钥后，平台会生成对应的支付宝公钥，保存此公钥用于验证回调签名

#### 5. 配置项说明

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `app_id` | 应用ID | 2021001234567890 |
| `private_key` | 应用私钥（PKCS8格式） | MIIEvQIBADANB... |
| `alipay_public_key` | 支付宝公钥 | MIIBIjANB... |
| `notify_url` | 异步通知地址（必须公网可访问） | https://api.example.com/api/v1/payments/alipay/callback |
| `return_url` | 同步跳转地址 | https://api.example.com/recharge |
| `sandbox` | 是否使用沙箱环境 | true/false |

### 沙箱环境

支付宝提供沙箱环境用于测试：
- 沙箱地址：https://openhome.alipay.com/develop/sandbox/app
- 沙箱版支付宝App：用于扫码支付测试

## 微信支付配置指南

### 前置条件

1. 注册微信支付商户平台：https://pay.weixin.qq.com/
2. 完成商户资质认证
3. 获取商户号（mchid）

### 配置步骤

#### 1. 获取商户号

登录微信支付商户平台 → 账户中心 → 商户信息 → 获取商户号

#### 2. 获取 AppID

在微信公众平台或微信开放平台获取 AppID

#### 3. 设置 APIv3 密钥

商户平台 → 账户中心 → API安全 → 设置APIv3密钥（32位）

#### 4. 下载证书

商户平台 → 账户中心 → API安全 → 申请证书 → 下载证书

#### 5. 配置项说明

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `mchid` | 商户号 | 1234567890 |
| `appid` | AppID | wx1234567890abcdef |
| `api_key` | APIv2 密钥（如果使用v2接口） | 32位随机字符串 |
| `cert_path` | API证书路径 | /path/to/apiclient_cert.p12 |
| `notify_url` | 支付结果通知地址 | https://api.example.com/api/v1/payments/wechat/callback |
| `sandbox` | 是否使用沙箱环境 | true/false |

### 沙箱环境

微信支付提供沙箱环境：
- 沙箱商户号与正式商户号不同
- 沙箱 API 密钥需要单独获取

## 回调通知配置

### 回调地址要求

1. **公网可访问**：第三方支付平台需要能访问到你的回调地址
2. **使用 HTTPS**：强烈建议使用 HTTPS
3. **保持幂等**：回调接口需要处理重复通知

### 回调验证

所有回调都需要验证签名：

#### 支付宝回调验证
```python
from alipay import AliPay

alipay = AliPay(...)
if alipay.verify(request_params, signature):
    # 验证通过，处理订单
```

#### 微信支付回调验证
```python
from wechatpayv3 import WeChatPay

wxpay = WeChatPay(...)
if wxpay.verify(request_params, signature):
    # 验证通过，处理订单
```

### 回调处理建议

1. 返回成功响应后再处理业务逻辑
2. 使用幂等键防止重复处理
3. 记录回调日志便于排查

## 配置安全建议

### 密钥存储

1. **不要硬编码**：将密钥存储在配置文件或环境变量
2. **加密存储**：启用 `is_encrypted` 选项
3. **定期轮换**：定期更换密钥

### 环境分离

| 环境 | 说明 |
|------|------|
| 开发环境 | 使用沙箱/测试环境 |
| 测试环境 | 使用沙箱/测试商户 |
| 生产环境 | 使用正式商户号 |

### 监控告警

建议配置以下告警：
- 支付成功率异常下降
- 重复回调通知
- 签名验证失败

## 常见问题

### Q1: 沙箱环境测试正常，但切换到生产环境失败

**检查项**：
1. 确认 AppID/MchID 是否正确
2. 确认公钥/密钥是否为生产环境生成
3. 确认 notify_url 是否为 HTTPS

### Q2: 回调通知接收不到

**检查项**：
1. 确认 notify_url 公网可访问
2. 检查防火墙/安全组设置
3. 查看服务端日志

### Q3: 签名验证失败

**检查项**：
1. 确认密钥格式正确（PKCS8）
2. 确认使用正确的支付宝公钥（不是应用公钥）
3. 检查编码格式（UTF-8）
