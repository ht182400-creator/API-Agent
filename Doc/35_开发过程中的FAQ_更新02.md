

## 问题125：Developer测试工具调用Weather API失败

### 125.1 问题描述

**发现时间**：2026-04-21 22:08

使用 Developer 测试工具调用 Weather API 时遇到多种错误：

**错误1：直接打开HTML文件时的 CORS 错误**
```
Access to fetch at 'http://localhost:8000/api/v1/...' from origin 'file:///...'
has been blocked by CORS policy
```

**错误2：file:// 协议安全问题**
```
unsafe attempt to load URL file:///... from frame with URL file:///...
'file:' URLs are treated as unique security origins.
```

**错误3：通过 API 平台调用返回 404**
```
{"detail":"Repository 'weather' not found"}
```

**错误4：测试工具显示的接口路径错误**
```
当前接口：GET /weather/current
```
实际应该是：
```
当前接口：GET /api/v1/weather/current
```

### 125.2 问题原因分析

#### 原因1：测试工具必须通过HTTP服务器访问

直接双击打开 HTML 文件会使用 `file://` 协议，浏览器会阻止向 `http://` 地址发送请求。

**解决方案**：必须启动 Python HTTP 服务器：
```bash
cd D:\Work_Area\AI\API-Agent\Developer\CallWeatherTest Tool\src
python -m http.server 8080
```
然后访问 `http://localhost:8080`

#### 原因2：API平台缺少动态路由

API 平台后端原本只支持固定的几个路由（如 `/chat`、`/translate`），不支持动态路径代理。

**解决方案**：添加了通用路由代理功能 `/{repo_slug}/{path:path}`

#### 原因3：weather-api仓库端点数据丢失

数据库中 weather-api 仓库的端点 `repo_id` 与仓库实际 `id` 不匹配，导致端点查询返回 0 条记录。

**解决方案**：运行数据库修复脚本重建端点数据

#### 原因4：weather-api仓库后端地址未配置

仓库的 `endpoint_url` 字段为空，导致请求无法代理到 Weather API (8001)。

**解决方案**：设置后端地址为 `http://localhost:8001`

#### 原因5：测试工具端点路径缺少前缀

测试工具硬编码的端点路径为 `/weather/current`，但 API 平台路由需要完整路径 `/api/v1/weather/current`。

**解决方案**：修改测试工具的端点配置

### 125.3 解决方案

#### 修复1：添加API平台动态路由代理

**修改文件**：`api-platform/src/api/v1/repositories.py`

在路由定义中添加通用路由代理：

```python
@router.api_route("/{repo_slug}/{path:path}", methods=["GET", "POST", "PUT", "DELETE", "PATCH"])
async def proxy_repository_endpoint(
    repo_slug: str,
    path: str,
    request: Request,
    x_access_key: str = Header(None, description="API Access Key"),
    db: AsyncSession = Depends(get_db),
):
    """
    通用仓库端点代理
    
    动态代理请求到仓库的实际后端服务。
    支持 GET、POST、PUT、DELETE、PATCH 方法。
    
    Args:
        repo_slug: 仓库 slug
        path: 仓库内部路径（如 weather/current）
    
    Returns:
        后端服务响应
    """
    import httpx
    from src.services.auth_service import AuthService
    
    # 1. 获取仓库信息
    repo_result = await db.execute(select(Repository).where(Repository.slug == repo_slug))
    repo = repo_result.scalar_one_or_none()
    
    if not repo:
        raise HTTPException(status_code=404, detail=f"Repository '{repo_slug}' not found")
    
    # 2. 检查仓库状态
    if repo.status != "online":
        raise HTTPException(status_code=403, detail="Repository is not online")
    
    # 3. 如果没有配置后端URL，返回模拟响应
    if not repo.endpoint_url:
        return BaseResponse(
            code=200,
            message="Success",
            data={
                "mock": True,
                "repo": repo.name,
                "path": f"/{path}",
                "message": "后端服务未配置，返回模拟响应",
            }
        )
    
    # 4. 验证 API Key
    auth_service = AuthService(db)
    user = None
    if x_access_key:
        try:
            user, _ = await auth_service.verify_api_key(x_access_key, repo_id=str(repo.id))
        except Exception:
            pass
    
    # 5. 转发请求到后端
    try:
        backend_url = f"{repo.endpoint_url.rstrip('/')}/{path}"
        query_params = dict(request.query_params)
        body = await request.body()
        
        async with httpx.AsyncClient(timeout=30.0) as client:
            headers = dict(request.headers)
            headers.pop("host", None)
            
            if x_access_key:
                headers["X-API-Key"] = x_access_key
            
            resp = await client.request(
                method=request.method,
                url=backend_url,
                params=query_params,
                content=body if body else None,
                headers=headers,
            )
            
            try:
                resp_data = resp.json()
                return resp_data
            except Exception:
                return {"status_code": resp.status_code, "content": resp.text}
                
    except httpx.TimeoutException:
        raise HTTPException(status_code=504, detail="Backend service timeout")
    except httpx.RequestError as e:
        raise HTTPException(status_code=502, detail=f"Backend service error: {str(e)}")
```

#### 修复2：创建端点修复脚本

**创建文件**：`api-platform/scripts/fix_weather_endpoints_sync.py`

```python
#!/usr/bin/env python
# -*- coding: utf-8 -*-
"""
修复 weather-api 仓库端点数据的脚本
"""

import sys
import os
import io

sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))))

from sqlalchemy import create_engine, select, delete
from sqlalchemy.orm import Session
from src.models.repository import Repository, RepoEndpoint

DATABASE_URL = os.getenv(
    "DATABASE_URL",
    "postgresql+psycopg2://postgres:postgres@localhost:5432/api_platform"
)


def main():
    sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')
    
    engine = create_engine(DATABASE_URL, echo=False)
    
    with Session(engine) as db:
        # 查找 weather-api 仓库
        result = db.execute(select(Repository).where(Repository.slug == "weather-api"))
        repo = result.scalar_one_or_none()
        
        if not repo:
            print("ERROR: 未找到 weather-api 仓库！")
            return
        
        repo_id = str(repo.id)
        
        # 删除旧端点
        db.execute(delete(RepoEndpoint).where(RepoEndpoint.repo_id == repo_id))
        
        # 创建正确的端点
        endpoints_data = [
            {"path": "/api/v1/weather/current", "method": "GET", "description": "获取当前天气"},
            {"path": "/api/v1/weather/forecast", "method": "GET", "description": "获取天气预报"},
            {"path": "/api/v1/weather/aqi", "method": "GET", "description": "获取AQI数据"},
            {"path": "/api/v1/weather/alerts", "method": "GET", "description": "获取预警信息"}
        ]
        
        for ep_data in endpoints_data:
            endpoint = RepoEndpoint(
                repo_id=repo_id,
                path=ep_data["path"],
                method=ep_data["method"],
                description=ep_data["description"]
            )
            db.add(endpoint)
        
        db.commit()
        print("OK: 端点修复完成！")


if __name__ == "__main__":
    main()
```

#### 修复3：设置后端地址脚本

**创建文件**：`api-platform/scripts/set_weather_backend_url.py`

```python
#!/usr/bin/env python
# -*- coding: utf-8 -*-
"""
设置 weather-api 仓库的后端地址
"""

import sys
import os
import io

sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))))

from sqlalchemy import create_engine, select, update
from sqlalchemy.orm import Session
from src.models.repository import Repository

DATABASE_URL = os.getenv(
    "DATABASE_URL",
    "postgresql+psycopg2://postgres:postgres@localhost:5432/api_platform"
)


def main():
    sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')
    
    engine = create_engine(DATABASE_URL, echo=False)
    
    with Session(engine) as db:
        result = db.execute(select(Repository).where(Repository.slug == "weather-api"))
        repo = result.scalar_one_or_none()
        
        if not repo:
            print("ERROR: 未找到 weather-api 仓库！")
            return
        
        # 设置后端地址
        backend_url = "http://localhost:8001"
        db.execute(
            update(Repository)
            .where(Repository.id == str(repo.id))
            .values(endpoint_url=backend_url)
        )
        db.commit()
        print("OK: 后端地址已设置为 http://localhost:8001")


if __name__ == "__main__":
    main()
```

## 问题126：Developer调用API后平台调用记录为0

### 126.1 问题描述

**发现时间**：2026-04-21 23:16

使用 Developer 测试工具成功调用了 Weather API 并获得正常响应，但在 API 平台的 Developer 调用记录页面显示调用次数为 0。

### 126.2 问题原因

API 平台后端的 `proxy_repository_endpoint` 函数在转发请求到仓库后端时，**没有记录 API 调用日志**。

虽然 `AuthService.verify_api_key` 函数有记录日志的功能，但实际转发逻辑（`proxy_repository_endpoint`）没有调用日志记录代码。

### 126.3 解决方案

**修改文件**：`api-platform/src/api/v1/repositories.py`

在 `proxy_repository_endpoint` 函数的 `finally` 块中添加 API 调用日志记录：

```python
finally:
    # 6. 记录 API 调用日志
    try:
        from src.models.billing import APICallLog, Quota
        from datetime import datetime, timedelta

        log_entry = APICallLog(
            repo_id=repo.id,
            api_key_id=key_obj.id,
            user_id=user.id,
            endpoint=f"/{path}",
            method=request.method,
            status_code=status_code,
            response_time=str(response_time),
            cost="0",
        )
        db.add(log_entry)

        # 更新每日配额使用量
        today_start = datetime.utcnow().replace(hour=0, minute=0, second=0, microsecond=0)
        quota_result = await db.execute(
            select(Quota).where(
                and_(
                    Quota.key_id == key_obj.id,
                    Quota.quota_type == "daily",
                    Quota.reset_at >= today_start
                )
            )
        )
        quota = quota_result.scalar_one_or_none()

        if quota:
            quota.quota_used += 1
        else:
            quota = Quota(
                user_id=user.id,
                key_id=key_obj.id,
                repo_id=repo.id,
                quota_type="daily",
                quota_limit=key_obj.daily_quota or 0,
                quota_used=1,
                reset_type="daily",
                reset_at=today_start + timedelta(days=1),
            )
            db.add(quota)

        await db.commit()
    except Exception as log_error:
        await db.rollback()
        print(f"Failed to log API call: {log_error}")
```

**关键修改点**：

1. 提取 `status_code`、`response_time`、`response_data` 到 `finally` 块外
2. 在 `finally` 块中创建 `APICallLog` 记录
3. 更新每日配额 `Quota` 表
4. 添加 `and_` 导入：`from sqlalchemy import select, func, and_`

## 问题127：配额使用情况RPM/RPH等数据为空

### 127.1 问题描述

**发现时间**：2026-04-22 00:04

在 Developer Portal 的「配额使用情况」页面，RPM/RPH 等数据为空或不显示。

| 指标 | 显示状态 |
|------|----------|
| 调用日志 | 正常 |
| 每日配额 | 正常 |
| 每月配额 | 正常 |
| 每分钟请求 (RPM) | 为空 |
| 每小时请求 (RPH) | 为空 |

### 127.2 问题排查过程

**第一步**：确认 API 返回数据 - 返回 rpm_used: 0, rph_used: 0

**第二步**：检查数据库 - 有 20 条记录，但时间是昨天 (2026-04-21)

**第三步**：发现时区问题
```python
datetime.now():    2026-04-22 00:14:16 (北京时间)
datetime.utcnow(): 2026-04-21 16:14:16 (UTC时间)
时区差异: 8小时
```

### 127.3 解决方案

#### 修复1：配额 API 时区统一

**文件**：`api-platform/src/api/v1/quota.py`

将 `datetime.now()` 改为 `datetime.utcnow()`

#### 修复2：RPM/RPH 统计优化

改为直接通过 `user_id` 统计

#### 修复3：日志查询表名修复

将 `/quota/logs` API 数据源从 `KeyUsageLog` 改为 `APICallLog`

### 127.4 数据计算规则

| 指标 | 计算规则 | 数据来源 |
|------|---------|---------|
| 每日配额 | Quota 表 quota_type='daily' | Quota 表 |
| 每月配额 | Quota 表 quota_type='monthly' | Quota 表 |
| RPM | 最近1分钟调用次数 | APICallLog 表 |

async def calculate_and_charge(
    db: AsyncSession,
    user_id: UUID,
    repo_id: UUID,

    """

    """

# 修改 get_consumption_trend 接口
# 从 APICallLog 统计调用次数 → 从 Bill 表统计消费金额

async def get_consumption_trend(...):
    # 查询消费账单并按日期聚合
    result = await db.execute(
        select(Bill).where(
            Bill.user_id == current_user.id,
            Bill.bill_type == "consume",
            Bill.created_at >= start_date,
        ).order_by(Bill.created_at)
    )
    bills = result.scalars().all()
    
    # 按日期聚合消费金额
    daily_data = {}
    for bill in bills:
        date_str = bill.created_at.strftime("%Y-%m-%d")
        if date_str not in daily_data:
            daily_data[date_str] = {"date": date_str, "amount": 0}
        daily_data[date_str]["amount"] += abs(float(bill.amount))
```

**文件**：`api-platform/src/api/v1/quota.py`

# 修改 get_usage_history 接口
# 从 KeyUsageLog 查询 → 从 APICallLog 统计

async def get_usage_history(...):
    # 从 APICallLog 表按日期聚合查询
    result = await db.execute(
        select(
            func.date(APICallLog.created_at).label("date"),
            func.count(APICallLog.id).label("call_count"),
            func.sum(func.cast(APICallLog.cost, Float)).label("total_amount")
        ).where(
            APICallLog.api_key_id == key_id,
            APICallLog.user_id == current_user.id,

        ).group_by(func.date(APICallLog.created_at))
    )
```
