## 第一章 概述

### 1.1 设计目标

本文档定义API服务平台仓库发布与审核的完整业务流程，确保：
- 任何用户都可以成为API提供者
- 平台API质量得到把控
- 管理员高效管理仓库审核
- 仓库所有者获得完整的收益管理功能

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 降低门槛 | 任何注册用户都能创建仓库 |
| 质量把控 | 管理员审核确保平台API质量 |
| 效率优先 | 管理员创建的仓库直接上线 |
| 动态菜单 | 根据用户是否有仓库显示对应功能 |

### 3.3 创建仓库逻辑

```python
async def create_repository(user, repo_data):
    """
    创建仓库逻辑
    
    规则：
    1. 管理员/超级管理员创建的仓库：status = "online"
    2. 普通用户/开发者创建的仓库：status = "pending"
    """
    if user.user_type in ['admin', 'super_admin']:
        status = "online"
    else:
        status = "pending"
    
    repo = Repository(
        name=repo_data.name,
        status=status,
        owner_id=user.id,
        owner_type="external"
    )
    
    # 发送通知（如果是pending状态）
    if status == "pending":
        notify_admins(f"新仓库待审核: {repo_data.name}")
    
    return repo
```

## 第六章 审核通知机制

### 6.1 通知触发场景

| 事件 | 通知对象 | 通知方式 |
|------|---------|---------|
| 新仓库待审核 | 管理员 | 系统消息 + 邮件 |
| 仓库审核通过 | 仓库所有者 | 系统消息 |
| 仓库审核拒绝 | 仓库所有者 | 系统消息 + 拒绝原因 |
