# 支付对账系统设计方案

## 一、需求背景

在真实支付模式下，平台需要与第三方支付渠道（支付宝、微信支付、银行等）进行资金核对，确保：
- 用户充值金额与平台入账金额一致
- 平台记录与第三方支付平台记录一致
- 发现异常交易及时处理

## 二、行业最佳实践

### 2.1 对账时间节点

| 类型 | 时间 | 说明 |
|------|------|------|
| 日对账 | 每日 T+1 凌晨 | 核对前一日交易 |
| 月对账 | 每月初 | 月度汇总结算 |
| 实时监控 | 7x24 | 异常交易告警 |

## 四、数据库设计

### 4.1 新增表结构

#### 4.1.1 平台账户表（platform_accounts）

```sql
CREATE TABLE platform_accounts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    channel VARCHAR(20) NOT NULL COMMENT '支付渠道: alipay/wechat/bankcard',
    account_type VARCHAR(20) NOT NULL COMMENT '账户类型: income(收入)/frozen(冻结)/available(可用)',
    account_no VARCHAR(100) COMMENT '第三方平台账户号',
    account_name VARCHAR(100) COMMENT '账户名称',
    balance DECIMAL(20,2) DEFAULT 0.00 COMMENT '账户余额',
    currency VARCHAR(10) DEFAULT 'CNY',
    status VARCHAR(20) DEFAULT 'active' COMMENT '状态: active/inactive',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_platform_accounts_channel ON platform_accounts(channel);
```

#### 4.1.2 对账记录表（reconciliation_records）

```sql
CREATE TABLE reconciliation_records (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    reconcile_date DATE NOT NULL COMMENT '对账日期',
    channel VARCHAR(20) NOT NULL COMMENT '支付渠道',
    platform_trade_count INT DEFAULT 0 COMMENT '本地交易笔数',
    platform_trade_amount DECIMAL(20,2) DEFAULT 0.00 COMMENT '本地交易金额',
    channel_trade_count INT DEFAULT 0 COMMENT '第三方交易笔数',
    channel_trade_amount DECIMAL(20,2) DEFAULT 0.00 COMMENT '第三方交易金额',
    match_count INT DEFAULT 0 COMMENT '匹配笔数',
    match_amount DECIMAL(20,2) DEFAULT 0.00 COMMENT '匹配金额',
    long_count INT DEFAULT 0 COMMENT '长款笔数(本地多)',
    long_amount DECIMAL(20,2) DEFAULT 0.00 COMMENT '长款金额',
    short_count INT DEFAULT 0 COMMENT '短款笔数(第三方多)',
    short_amount DECIMAL(20,2) DEFAULT 0.00 COMMENT '短款金额',
    amount_diff_count INT DEFAULT 0 COMMENT '金额不一致笔数',
    amount_diff_total DECIMAL(20,2) DEFAULT 0.00 COMMENT '金额差异总额',
    status VARCHAR(20) DEFAULT 'pending' COMMENT '对账状态: pending/completed/disputed',
    bill_file_path VARCHAR(500) COMMENT '第三方账单文件路径',
    operator_id UUID REFERENCES users(id),
    completed_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_reconcile_records_date ON reconciliation_records(reconcile_date);
CREATE INDEX idx_reconcile_records_channel ON reconciliation_records(channel);
CREATE UNIQUE INDEX idx_reconcile_records_date_channel ON reconciliation_records(reconcile_date, channel);
```

#### 4.1.3 对账差异明细表（reconciliation_disputes）

```sql
CREATE TABLE reconciliation_disputes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    reconciliation_id UUID REFERENCES reconciliation_records(id),
    dispute_type VARCHAR(20) NOT NULL COMMENT '差异类型: long/short/amount_diff',
    local_order_no VARCHAR(100) COMMENT '本地订单号(长款时必填)',
    channel_trade_no VARCHAR(100) COMMENT '第三方交易号(短款时必填)',
    local_amount DECIMAL(20,2) COMMENT '本地金额',
    channel_amount DECIMAL(20,2) COMMENT '第三方金额',
    diff_amount DECIMAL(20,2) COMMENT '差异金额',
    reason VARCHAR(500) COMMENT '差异原因',
    handle_status VARCHAR(20) DEFAULT 'pending' COMMENT '处理状态: pending/confirmed/resolved/ignored',
    handle_remark VARCHAR(500) COMMENT '处理备注',
    handler_id UUID REFERENCES users(id),
    handled_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_disputes_recon_id ON reconciliation_disputes(reconciliation_id);
CREATE INDEX idx_disputes_status ON reconciliation_disputes(handle_status);
```

所有对账操作需要记录：
- 操作人
- 操作时间
- 操作内容
- 操作前后数据
- IP地址

## 九、实施方案建议

### 阶段一：基础功能（P0）
1. 充值明细查询页面
2. 渠道收款汇总
3. 平台账户余额展示

### 阶段二：对账核心（P1）
1. 手动对账功能
2. 对账结果展示
3. 差异记录

### 阶段三：自动化（P2）
1. 定时对账任务
2. 差异自动处理规则
3. 对账报表导出

### 阶段四：高级功能（P3）
1. 对账预警通知
2. 对账趋势分析
3. 智能差异分析

## 十、注意事项

1. **对账时机**：建议在 T+1 凌晨执行，避免影响日间交易
2. **数据安全**：账单文件需加密存储，设置访问权限
3. **异常处理**：网络异常时支持断点续传
4. **历史保留**：对账记录至少保留 5 年
5. **权限隔离**：财务人员与操作人员权限分离
