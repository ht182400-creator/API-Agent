## 第三章 Windows 环境搭建（详细步骤）

### 3.1 启用 WSL 2

WSL 2（Windows Subsystem for Linux）是Windows下开发Docker应用的最佳方式。

#### 步骤一：以管理员身份打开PowerShell

1. 点击开始菜单
2. 搜索 "PowerShell"
3. 右键选择 "以管理员身份运行"

#### 步骤二：启用所需Windows功能

```powershell
# 启用WSL
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

# 启用虚拟机平台
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
```

#### 步骤三：设置WSL默认版本

```powershell
# 设置默认版本为WSL 2
wsl --set-default-version 2
```

#### 步骤四：安装Ubuntu

1. 打开Microsoft Store
2. 搜索 "Ubuntu 22.04 LTS"
3. 点击安装
4. 首次启动时设置用户名和密码

#### 步骤五：验证安装

```powershell
# 查看已安装的WSL
wsl --list --verbose

# 预期输出：
# NAME                   STATE           VERSION
# Ubuntu-22.04           Running         2
```

### 3.2 安装Docker Desktop

#### 步骤一：下载Docker Desktop

访问 https://www.docker.com/products/docker-desktop 下载Windows版本。

#### 步骤二：安装Docker Desktop

1. 双击下载的安装包 `Docker Desktop Installer.exe`
2. 勾选 "Use WSL 2 instead of Hyper-V"（推荐）
3. 点击 "Install" 开始安装
4. 安装完成后点击 "Close and Restart"

#### 步骤三：配置Docker Desktop

1. 启动Docker Desktop
2. 点击右上角设置图标（⚙️）
3. 进入 "General" 设置：
   - 勾选 "Use the WSL 2 instead of Hyper-V"
4. 进入 "Resources" 设置：
   - WSL Integration：启用Ubuntu集成
   - Advanced：分配CPU和内存（建议4核CPU，4GB内存）
5. 点击 "Apply & Restart"

#### 步骤四：验证Docker安装

```powershell
# 在PowerShell中测试
docker --version
# 预期输出：Docker version 24.x.x

docker-compose --version
# 预期输出：Docker Compose version v2.x.x

# 运行hello-world测试
docker run hello-world
```

### 3.3 安装VS Code和必要插件

#### 步骤一：下载安装VS Code

访问 https://code.visualstudio.com/ 下载并安装。

#### 步骤二：安装中文语言包

1. 打开VS Code
2. 按 Ctrl+Shift+P 打开命令面板
3. 输入 "Configure Display Language"
4. 选择 "Install Additional Languages"
5. 选择 "Chinese (Simplified)" 安装

#### 步骤三：安装开发插件

推荐安装以下插件，可在VS Code扩展商店搜索安装：

| 插件名称 | 插件ID | 用途 |
|----------|--------|------|
| Python | ms-python.python | Python语言支持 |
| Pylance | ms-python.vscode-pylance | Python类型检查 |
| Docker | ms-azuretools.vscode-docker | Docker集成 |
| REST Client | humao.rest-client | API测试 |
| GitLens | eamodio.gitlens | Git增强 |
| Python Docstring | njpwerner.autodocstring | 文档生成 |
| YAML | redhat.vscode-yaml | YAML编辑 |
| JSON | vscode.json-language-features | JSON支持 |
| Thunder Client | rangav.vscode-thunder-client | API客户端 |

#### 步骤四：配置VS Code

创建或编辑 `.vscode/settings.json`：

```json
{
    "python.defaultInterpreterPath": "python",
    "python.linting.enabled": true,
    "python.linting.pylintEnabled": true,
    "python.formatting.provider": "black",
    "python.testing.pytestEnabled": true,
    "files.exclude": {
        "**/__pycache__": true,
        "**/*.pyc": true,
        "**/.pytest_cache": true
    },
    "editor.formatOnSave": true,
    "editor.rulers": [88, 120],
    "files.trimTrailingWhitespace": true,
    "docker.dockerPath": "docker"
}
```

### 3.4 配置Git

#### 步骤一：安装Git for Windows

从 https://git-scm.com/download/win 下载安装。

#### 步骤二：配置Git

```powershell
# 设置用户名和邮箱（与GitHub账号一致）
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# 设置默认分支名为main
git config --global init.defaultBranch main

# 启用颜色输出
git config --global color.ui auto

# 设置默认编辑器为VS Code
git config --global core.editor "code --wait"

# 设置换行符处理（Windows推荐）
git config --global core.autocrlf true
```

#### 步骤三：生成SSH密钥（可选）

```powershell
# 生成SSH密钥
ssh-keygen -t ed25519 -C "your.email@example.com"

# 查看公钥
cat ~/.ssh/id_ed25519.pub
```

将公钥添加到GitHub/GitLab账户。

## 第四章 macOS 环境搭建（详细步骤）

### 4.1 安装Homebrew

Homebrew是macOS的包管理器，简化软件安装过程。

```bash
# 安装Homebrew（可能需要输入密码）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装完成后配置PATH（Apple Silicon Mac）
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

# 验证安装
brew --version
```

### 4.2 安装Docker Desktop

```bash
# 方式一：使用Homebrew安装
brew install --cask docker

# 方式二：从Docker官网下载安装
# 访问 https://docs.docker.com/desktop/install/mac-install/
```

安装完成后：
1. 打开Applications文件夹
2. 双击Docker图标启动
3. 首次运行需要在系统偏好设置中授权

### 4.3 安装开发工具

```bash
# 安装Git
brew install git

# 安装Python（通过pyenv管理多版本）
brew install pyenv pyenv-virtualenv

# 配置pyenv（添加到shell配置）
echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.zshrc
echo 'command -v pyenv > /dev/null || export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.zshrc
echo 'eval "$(pyenv init -)"' >> ~/.zshrc
echo 'eval "$(pyenv-virtualenv-init -)"' >> ~/.zshrc

# 使配置生效
source ~/.zshrc

# 安装Python 3.11
pyenv install 3.11.0
pyenv global 3.11.0

# 验证
python --version
```

### 4.4 安装VS Code

```bash
# 使用Homebrew安装
brew install --cask visual-studio-code

# 或者从官网下载：https://code.visualstudio.com/
```

## 第五章 Linux 环境搭建（详细步骤）

### 5.1 Ubuntu/Debian 系统

#### 安装Docker Engine

```bash
# 更新apt包索引
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg

# 添加Docker官方GPG密钥
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# 设置Docker仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
    sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装Docker
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 将当前用户加入docker组（免sudo）
sudo usermod -aG docker $USER
newgrp docker

# 验证安装
docker run hello-world
```

#### 安装Python开发环境

```bash
# 安装Python和pip
sudo apt-get update
sudo apt-get install python3.11 python3-pip python3.11-venv python3.11-dev

# 安装常用开发工具
sudo apt-get install build-essential libpq-dev

# 验证
python3 --version
pip3 --version
```

#### 安装VS Code

```bash
# 下载VS Code deb包
wget -qO- https://packages.microsoft.com/keys/microsoft.asc | gpg --dearmor > packages.microsoft.gpg
sudo install -o root -g root -m 644 packages.microsoft.gpg /usr/share/keyrings/
sudo sh -c 'echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/packages.microsoft.gpg] https://packages.microsoft.com/repos/code stable main" > /etc/apt/sources.list.d/vscode.list'

# 安装VS Code
sudo apt-get update
sudo apt-get install code
```

### 5.2 CentOS/RHEL 系统

#### 安装Docker

```bash
# 安装依赖
sudo yum install -y yum-utils

# 设置Docker仓库
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# 安装Docker
sudo yum install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 启动Docker
sudo systemctl start docker
sudo systemctl enable docker

# 验证
sudo docker run hello-world
```

## 第七章 启动开发服务

### 7.1 启动依赖服务（Docker Compose）

使用Docker Compose启动数据库、Redis、MinIO等依赖服务：

```bash
# 进入docker目录
cd docker

# 启动所有依赖服务
docker-compose up -d postgres redis minio

# 查看服务状态
docker-compose ps

# 查看服务日志
docker-compose logs -f postgres

# 停止服务
docker-compose down

# 停止并删除数据卷（慎用，会清空数据）
docker-compose down -v
```

### 7.2 初始化数据库

```bash
# 创建数据库
docker exec -it api-postgres createdb -U api_user api_service

# 方式一：使用初始化脚本（推荐）
python scripts/init_db_with_data.py --drop  # 完整初始化，包含测试数据

# 方式二：仅初始化基础数据
python scripts/init_db.py

# 方式三：手动执行迁移脚本
python migrations/add_user_fields.py          # 用户字段扩展
python migrations/add_encrypted_key.py         # API Key 加密字段
```

> **注意**：每次拉取新代码后，建议检查 `migrations/` 目录是否有新的迁移脚本需要执行。

### 7.3 启动API服务

#### 方式一：直接运行

```bash
# 激活虚拟环境
source venv/bin/activate

# 启动服务
python src/main.py
# 或
uvicorn src.main:app --reload --host 0.0.0.0 --port 8080
```

#### 方式二：使用FastAPI开发服务器

```bash
# 安装uvicorn（如果未安装）
pip install uvicorn[standard]

# 以开发模式启动（热重载）
uvicorn src.main:app --reload --host 0.0.0.0 --port 8080 --log-level debug
```

#### 方式三：使用PyCharm运行配置

1. 打开PyCharm
2. File -> Settings -> Project -> Python Interpreter
3. 选择项目虚拟环境
4. 添加运行配置：
   - Script path: `src/main.py`
   - Parameters: `--reload --host 0.0.0.0 --port 8080`
   - Working directory: 项目根目录

```
Port: 5432
Database: api_service
User: api_user
Password: your_password
```

### 8.2 Redis连接

#### 使用RedisInsight

1. 下载 RedisInsight: https://redis.com/redis-enterprise/redis-insight/
2. 添加连接：
   - Host: localhost

## 第十章 常见问题与解决

### 10.1 Docker相关问题

#### Q1: Docker Desktop启动失败（Windows）

**症状**：Docker Desktop无法启动或报错。

**解决方案**：
1. 检查BIOS中是否启用了虚拟化技术
2. 重新安装Docker Desktop（先卸载）
3. 启用Windows功能：
   ```powershell
   dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
   dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
   ```
4. 重启电脑

#### Q2: WSL2内存占用过高

**解决方案**：在 `.wslconfig` 文件中限制内存：
```powershell
# 创建配置文件
notepad $env:USERPROFILE\.wslconfig
```

内容：
```ini
[wsl2]
memory=4GB
processors=2
```

#### Q3: Docker镜像拉取慢

**解决方案**：
1. 配置Docker镜像加速：
   ```json
   {
     "registry-mirrors": [
       "https://mirror.ccs.tencentyun.com",
       "https://docker.nju.edu.cn"
     ]
   }
   ```
2. 或者使用代理

### 10.2 Python相关问题

#### Q4: pip安装失败

**解决方案**：
```bash
# 升级pip
python -m pip install --upgrade pip

# 使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

#### Q5: 虚拟环境激活失败（Windows PowerShell）

**解决方案**：
```powershell
# 允许执行脚本
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 重新激活
.\venv\Scripts\Activate.ps1
```

#### Q6: 找不到Python解释器

**解决方案**：
1. VS Code: Ctrl+Shift+P -> Python: Select Interpreter
2. 选择项目虚拟环境中的Python

### 10.3 数据库相关问题

#### Q7: 数据库连接被拒绝

**解决方案**：
1. 检查PostgreSQL是否运行：
   ```bash
   docker ps | grep postgres
   ```
2. 检查连接信息是否正确
3. 检查防火墙设置

#### Q8: 数据库迁移失败

**解决方案**：
```bash
# 查看可用的迁移脚本
ls migrations/

# 手动执行迁移脚本
python migrations/add_user_fields.py
python migrations/add_encrypted_key.py

# 重置数据库（仅开发环境，慎用！）
python scripts/init_db_with_data.py --drop
```

### 10.4 网络相关问题

#### Q9: localhost无法访问

**解决方案**：
1. 检查服务是否绑定到正确地址（0.0.0.0而非127.0.0.1）
2. 检查端口是否被占用：
   ```bash
   netstat -ano | findstr :8080
   ```

#### Q10: SSL证书错误

**解决方案**：如果是自签名证书，浏览器访问时需要手动信任：
1. 点击"高级"
2. 点击"继续前往"
